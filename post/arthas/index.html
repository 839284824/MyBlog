【Arthas】实战

2022-03-18 4 min read # 实战 # Java

Arthas是什么？

Arthas 是Alibaba开源的Java诊断工具

  • https://github.com/alibaba/arthas/blob/master/README_CN.md
  • https://arthas.aliyun.com/doc/logger.html

Arthas可以解决哪些问题？

  • 这个类从哪个 jar 包加载的？为什么会报各种类相关的 Exception？
  • 我改的代码为什么没有执行到？难道是我没 commit？分支搞错了？
  • 遇到问题无法在线上 debug，难道只能通过加日志再重新发布吗？
  • 线上遇到某个用户的数据处理有问题，但线上同样无法 debug，线下无法重现！
  • 是否有一个全局视角来查看系统的运行状况？
  • 有什么办法可以监控到JVM的实时运行状态？
  • 怎么快速定位应用的热点，生成火焰图？
  • 怎样直接从JVM内查找某个类的实例？

Artha常用命令使用

dashboard 当前系统的实时数据面板信息。具体信息参考

  • https://arthas.aliyun.com/doc/dashboard

thread -n 3 一目了然的了解系统的状态，哪些线程比较占cpu？他们到底在做什么？

jvm 查看当前JVM信息
jad javax.servlet.Servlet 反编译类
memory 查看内存信息
logger 查看日志信息
sc 查看JVM已加载的类信息
vmtool vmtool 利用JVMTI接口，实现查询内存对象，强制GC等功能。神特性

monitor 方法执行监控,可以查看方法的成功率和RT信息。非常有用

watch 函数执行数据观测。可以看到方法的执行时的参数、返回值、是否异常等信息

  • https://arthas.aliyun.com/doc/watch.html

特殊用法：

  • https://github.com/alibaba/arthas/issues/71
    按照参数过滤
    watch com.alibaba.youku.search.service.service.impl.SearchServiceImpl query '{params,returnObj,throwExp}' 'params[0].get("appScene").equals("mobile_multi")' -n 5 -x 3 -v

trace 观察方法执行的时候哪个子调用比较慢。非常有用

  • https://arthas.aliyun.com/doc/trace.html#id1

stack 输出当前方法被调用的调用路径

  • https://arthas.aliyun.com/doc/stack.html

Time Tunnel(tt) 记录方法调用信息，支持事后查看方法调用的参数，返回值，抛出的异常等信息，仿佛穿越时空隧道回到调用现场一般。记录现场，并支持回放功能非常有用

  • https://arthas.aliyun.com/doc/tt.html

Profiler 生成火焰图.支持的事件用 list 查看,用--event指定不同的事件

  • https://arthas.aliyun.com/doc/profiler.html
